Introduction to Github

What is Github?

GitHub is a mostly free platform that developers use to store and manage their code online. It is mainly used for the following purposes:

  • Version Control: GitHub uses Git, a tool that helps track changes in code, making it easy to keep different stated of the project and recover if needed.
  • Collaboration: Multiple people can work on the same project at the same time without messing up each other’s work.
  • Backup and Sharing: Since Github stores your code online, you can keep a copy where you can share or keep private.
  • Open Sorce: For Open Source projects, where anyone can contribute, it's a great place to share the code.
  • Go to Github

What is a Repository?

  • A repository (or "repo") on GitHub is essentially a folder that holds all the files and version history for a specific project.
  • It’s a place where you store, manage, and track your code and other project files.
  • For this class it's where you are going to store your assignment files.
  • You will create 1 repo for every assignment you do using HTML.

Common Repository Terms

  • Clone: When you copy a repository from GitHub to your computer so you can work on it offline.
  • Push: When you upload changes from your local machine to the repository on GitHub
    • \
  • Pull: When you download changes from the GitHub repository to your computer.
  • Fork: When you create your own copy of someone else’s repository so you can make changes without affecting the original.

Creating a Repository

Once you login, create a new repo.

Make your repo public
Name your repo assignment-1
Add a README file

Click on Create Repository

At the top left of the page click on the burger menu, select Home and your repo should be on the left column.

Adding your Repo to Github Desktop

You have an online repository on Github, but you are working on your computer.
You need to syncronize your online files with your computer files, to make them available online.

On Github, select the code button and select
Open with Github Desktop

On Github Desktop, you need to add the location of the folder where you want to have your files in your computer.
Then click Clone. You might have to login to your Github account from Github Desktop.

The next screen at the top will have the current repository at the top left.

Updating your Files

You are going to Use Visual Studio Code to update your files, but before you do, you have to make sure Visual Studio is the default Editor on Github Desktop.

Go to Github Desktop and select Settings

On Integrations, make sure Visual Studio Code is selected.

Updating files on Visual Studio Code

Now that you created a copy of your repo on your computer, you need to open it on Visual Studio Code.

Select New Window, then select the repo folder
that was just copied to your computer.

Visual Studio Code will ask you
if your trust the files on the folder. Click Yes.

Updating files on Visual Studio Code

Once in Visual Studio code, create your files and folders for your assignment.

If you go to Github Desktop you will see a list of the changes you made.

Add a Comment on the Summary field on the bottom right and click Push Origin to upload your files to Github.
