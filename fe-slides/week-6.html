Conditional if/else statements

Conditional statements determine when and where you execute code, based on specific conditions.

  • if (condition) {
    // We do something if the condition is true
    } else {
    // The condition is not true and we do something else
    }
  • In order to evalueate if a condition is true or false, JavaScript relies on the following:
  • Comparison Operators.
  • Truthy and falsey values.

Comparison Operators

Comparison Operators always return a boolean. true or false

  • > greater than.
  • >= greater or equal to.
  • < less than.
  • <= less or equal to.
  • == equal === strictly equal.
  • !=unequal !== strictly unequal.

Equal or Strictly Equal

==
  • Looks for value equality only.
  • In that case a string and a number with the same output would be considered equal
  • "24" == 24 That would be true.
===
  • Makes sure that the elements compared are the same type and have the same value.
  • "24" === 24 That would be false.
  • 24 === 24 That would be true.

Truthy and Falsey

A truthy value is everything that evaluates to true in a conditional. Javascript coerces very value that would be a true or false statement into a boolean.

Falsey Values

  • false
  • null
  • undefined
  • 0 (the number 0)
  • NaN
  • empty string

Truthy Values

  • "0" (a string with the number 0)
  • "false" (a string with the word false)
  • [] (an empty array)
  • {} (and empty object)
  • function(){} (an empty function)
  • Using them you can reduce the amount of code you write.
  • Instead of writing if (myVar === null || myVar === undefined){} you can write if (myVar) {}
  • Javascript will get all the options and will give you a true or false response.

Null and Undefined

They may look similar but they are not the same.

Null

  • Null is an absence of value
  • You get Null when the element is empty
  • You can also declare a value to be null. middleName: null

Undefined

  • Is a value that has not been explicitly defined.
  • A variable that has no assignment will be undefined.
  • A function that has no return will also be undefined

BOM and DOM

BOM

    and refers to all the properties and methods available for JavaScript to interact with the web browser.
  • Unlike the DOM there is no standard definition and it's implemented differently depending on the browser.
  • The BOM main task is to manage the browser window.
  • Each time you load a page on the browser, it becomes a Document object.
  • DOM is a subset of BOM.

DOM

  • The DOM stands for the Document Object Model.
  • DOM is an interface for representing an interacting with the HTML.
  • In the DOM tree structure, every element is defined as an object, whith properties and methods.
  • We can interact with the DOM with any programming language, but it's typically used with Javascript.

DOM Objects

The DOM's purpose is to provide Javascript with a map of all the elements in the page, and provide us a set of methods to access those elements.

  • window: Represents the open window that contains the DOM document. It contains all the Javascript functions and methods. document.head / body
  • document: Represents the HTML document in the window. console.log(document)
  • history: Previous URL's visited by the client. Useful for using back and forward methods. window.history.back()
  • location: Information about the current URL
  • navigator: Information about the client. (Browser)

Selecting an element with Javascript

The document object has a method called querySelector. This returns the first Element within the document that matches the specified selector, or group of selectors. If no matches are found, null is returned.

const selectElement = document.querySelector('selector')

  • selector is the class, id, html tag or attribute we are targeting.
  • You use the same notation you use in CSS to refer to those elements:
    • id: #
    • class: .
    • tag: p
    • attribute: [data-type="type"]

ID, class, attributes or tags.

  • # Id's are good when you want to target one specific element in the page.
  • . Classes are good when you want to select more than one element. You can target single elements too, but run the risk to have the class added to another element.
  • [] Attributes are an additional method to store information and pass it to Javascript. They are also used when you want to target more than one element that share the same data type, but it can be also used for single elements.
  • p It is too generic, and unless you want to target all instances of that particular element, it is better to use the other options.

Options to querySelector

  • The querySelector lets you select an element from the DOM using its CSS selector.
  • getElementById Works the same but its limited to selecting id's only.
  • getElementsByClassName Allows you to target elements using CSS classes only.
  • getElementsByTagName lets you select HTML tags.
  • Try to choose querySelector. It is newer and more flexible, but if you choose the other options, be consistent.
  • querySelector lets you target data-types, tags or you need to use complex selectors. document.querySelector('div#idName')
  • querySelectorAll lets you access all elements of that particular type. This will return an Array.

Arrays

Arrays are a little like variables, they hold content. The difference is that a variable holds one element, an Array holds a list of elements.

  • An array is created by using [] and separating the elements with commas.
  • const myArray = [item1, item2, item2]
  • Arrays can contain any type of data: strings, numbers, objects, functions and even other arrays.
  • If the array is short, it can be on one line, if it gets too long it can go in different lines.
  • Javascript arrays are zero-based indexes because the count starts at 0.

Array Methods

Arrays are great to give us list of things. But in order to interact with them we need to know its position in the array. We also need to be able to add/delete elements to the array.

  • indexOf gets you the position of an element in the array. I can only be used with primitives.
  • findIndex works the same as indexOf but for abjects, arrays or functions. It works differently than indexOf because it executes a function for every array element.
  • unshift adds elements to the beginning of the array.
  • push adds elements to the end of the array.
  • shift lets you remove an element from the start.
  • pop lets you remove an item from the end.
  • splice lets you add/remove elements anywhere in the array and you can do it more than one item at a time.

Splice

const deletedItems = array.splice(index, deleteCount, itemsToAdd)

  • index is the position of the element where you want to start.
  • deleteCount is the number of items you want to delete. It can be 0 if you jusr want to add.
  • itemsToAdd items you want to add.
  • You can use splice to add or delete elements at the start or end, but it's easier to use unshift and push.
  • slice Not to be confused with splice lets you create a copy of an array.

Loops

JavaScript loops are used to create repetitive tasks.

  • Loops validate a particular value against a test condition and execute a statement for a set amount of times.
  • There are different loop types in Javascript:
    • For Loops
    • While loops
    • Do-while loops
    • For-in loops
    • Break statements
    • Continue statement

For Loops

A for loop runs a block of code a number of specified times.

  • for (initialExpression; condition; incrementExpression) {
    statement
    }
  • The initialExpression is the initial condition. In most cases is called i and is equal to 0. let i = 0
  • The incrementExpression increments the value until the condition is met.
  • The condition is determines the maximum amount of times the statement has to run.
  • The statement is the code you want to run multiple times if the condition is true.
  • If the condition is false the loop is skipped.
  • for (let i = 0; i < 10; i++)

Infinite loops

    There are some occasions in which the condition of the loop will always evaluate as true.
  • for (let i = 0; i < 1; i--) {
    console.log('I'll run forever')
    }
  • To end an infinite loop you have to close the browser. There is no way to stop it.

Using Conditionals in loops

Most of the time you will be writing loops to get elements inside an array. You can use any form of conditionals inside a loop.

  • const dollarBills = [1, 5, 20, 50, 100, 500, 1000, 2000];

The for-of loop

This is a loop introduced with ES6. It is better than a for loop because it is easier to read and maintain.

  • for (let bill of dollarBills) {
    if (bill < 101) {
    console.log(bill + ' is a valid dollar bill!')
    }
    else{
    console.log(bill + ' is NOT a valid dollar bill!')
    }
    }

The foreach loop

All arrays contain a forEach method that helps you loop through every item.

  • dollarBills.forEach(function(bill){
    if (bill < 101) {
    console.log(bill + ' is a valid dollar bill!')
    }
    else{
    console.log(bill + ' is NOT a valid dollar bill!')
    }
    })
  • for of and foreach are better to use than the regular for loop.

Javascript and CSS

You can change any css property on an element with Javascript. There are different ways to do it.

The style property

  • You can change any value with the following statement:
  • Element.style.cssProperty = propertyValue
  • You can use any css property but it it has to words, it needs to be written in CamelCase.
  • element.style.fontSize = "2rem"

the setProperty

This method takes 2 values. The property and the value, and it can be used to create custom properties (--myColor)

  • element.style.setProperty('color', 'red')

The setAttribute

You can also add data-attributes to html element with Javascript.

  • Element.setAttribute('name-of-attribute', 'value to set')

Events

When a user clicks on a button, that's an event. We need to register when these events happen. We do it with Event listeners. You can add them to any element in the DOM.

  • Element.addEventListener('event-name', function () {
    })
  • the "event-name" is the name of the event you want to listen to. (click, scroll, etc) Ecent Reference List.
  • The function is the code you want to execute when the event happens.
  • You can check if an event exists by looking at the inspector. It should have an "event" label in Firefox and it displays the function on Chrome when clicked (Use $0 on the console).
