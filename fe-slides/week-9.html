React

React is a Javascript library created by Facebook in 2013 to accomodate fo a scalable and ever changing interface.

React forces you to think in terms of component, and modular elements.

React's goal is to simplify the development of user interfaces.

React is the most popular Javascript library used today.

What do I need to start with React?

In order to work with React you need an understanding of:

  • HTML
  • CSS
  • JavaScript
    • Variables
    • Arrow Functions
    • Objects and Arrays

How do I install React?

The best way to install React is by using the create-react-app on the command line.

We have to check a few things first:

  • You need to have node and npm installed

Why Node.js?

  • Node.js is a runtime environment for JavaScript that runs on the server.
  • A runtime environment is an environment in which a program or application can be executed.
  • Traditionally, Javascript run in the browser, for client-side scripting using the Script tag.
  • Node is not a programming language nor a framework. It's an environment for back-end javascript applications.
  • Check if you have node installed by typing node -v on the terminal.
  • Download Node.js and follow installation instructions.

Installing React

  1. Once you check you have node installed, navigate to the folder where you want to create your application by typing cd yourFolder on the terminal prompt.
  2. Once in the folder, type npx create-react-app my-app-name.
  3. Once the application is created, open the folder on Visual Studio Code and open a terminal panel. This terminal will be located inside the recently created application.
  4. Then run the application by typing npm start on the terminal.
  5. A new browser window will open at http://localhost:3000/. React will open a temporsry server to run the app on your computer.

Basic React Concepts

  • Components: Components are re-usable elements or building blocks. The idea is to keep smaller chuncks of code in different components, rather than having a function that does many things at the same time.
  • JSX: It's a syntax extension to JavaScript used by REACT. It is basically HTML code inside Javascript. Regularly, we cannoy use plain HTML inside Javascript, but with REACT, we can and it will transformed to proper Javascript syntax.
  • State: The state is a built-in React object used to contain data or information about the component. A component's state can change over time; whenever it changes, the component re-renders.
  • Props: Props are arguments passed into React components.

More on JSX

JSX was introduced by React. You don't need to use JSX to work in react, but it's a lot easier if you do.

There are a few rules: When an attribute has a - it will be automatically converted into camelCase.

When using CSS, class is a reserved word for JavaScript, so we use className instead.

The same goes for the html for like in label for= where we have to use htmlFor instead.

In JSX, all tags need to be closed <br> needs to be <br />

Navigating the React folder structure

When you install React, you have 3 main folders: node_modules, public and src.

  • node_modules: When you install React using create-react-app it loads many modules you can import into your projects.
  • public: This folder contains the html file, and it can also contain other assets. Files inside the Public folder will not be processed by webpack. (a module bundler for javascript). These files will be untouched on the build process. You can rename/delete all the files in this folder except for the index.html.
  • src: This is where all the files will be processed by webpack. Files outside this folder will not be included in the build process. All Javascript and CSS need to live in this folder. You can rename all the files except for the index.js
