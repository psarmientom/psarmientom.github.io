CSS Units

px, em, rem, ch, vh, vh

  • Why should we use em or rem instead of px?
  • px are fixed units that will override any custom settings, either by the user or on the stylesheet.
  • em is relative to the parent value.
  • rem is relative to the root value.
  • Pixel to em conversion. | Pixel to em calculator.
  • You can also set the root font-size to 62.5% (10px) which makes the calculation easier.
  • ch Limits the width of an element to the selected amount or characters regardless of the font.
  • Ideal number of characters is 60 and 100 characters per line including spacing

Responsive Images

We all know that images are the biggest offender for page size. Here are a few techniques to optimize.

  • Size your images closer to the size in which they are going to be displayed.
  • For full width images use 1920 x 1080, or at least 1920px wide.
  • Make sure to add the alt="" description.
  • Add the width and height. width="1920" height="1080"
  • Use lazy loading. loading="lazy" to load images only when entering the viewport.
  • For bigger images, consider using a smaller version of the same image so the browser can choose which one to use.
  • Use Responsive Images.

Advanded CSS Selectors

Applying css styles to html elements, id's and classes is the simplest choice. But there are many other ways we can target specific elements without touching the html.

  • * The Universal selector. Applies to every element in the document.
  • > Direct Descendant (only parent-child relationship).
  • + Immediate adjacent sibling.
  • ~ Any sibling. (tilde) When working with siblings the effect applies to any element after the targeted one, not before.
  • [href] [name] [title] a[class] etc. Targets the attribute. Attribute list.
  • [title~="myClass"] Using ~ in combination with the attribute class, looks for any element that contains the give attribute.
  • [title^="my"] Select all the elements that start with the attribute.
  • [title$="Class"] Select all the element that end with the attribute.
  • [title*="Cl"] Selects all the elements that contain the attribute.

Pseudo Classes

Pseudo Classes are keywords that are applied when they match the selected state or context of an element.

  • :link, :visited, :focus, :hover, :active Links pseudo classes. They should be added in this irder in order to maintain the cascade.
  • :only-child Selects the only child of that element in relation to the parent.
  • :only-of-type Selects the only child of that element type.
  • :first-child :last-child Selects the first or last child on a group or list.
  • :nth-child(2) :nth-child(odd) Selects the specific element on a list of siblings.
  • :nth-child(3n +2) Selects every 3 elements starting on the second one. li:nth-last-child(3n + 2) Selects from the bottom up.
  • :target Used with in-page navigation, select the targeted area.
  • :not([attribute]) :not(.class) :not(div) Select the element that do NOT match the selector.
  • img:not([alt]) {border:solid 5px red} Helps you check which images do not have an alt text.
  • :root Selects the root element of the document, similar to the html selector.
  • It is commonly used to set the css variables.
  • :empty When targeting a tag that has no content or white space.
  • Mostly used with scripting languages to style or target dynamic content.

Pseudo Elements

The Pseudo Element is a keyword added to a selector that lets you style a specific part of the selected element.
Pseudo-elements have :: instead of one. Pseudo classes have one :

  • :before :after Add styling before or after an element.
  • ::first-line ::first-letter | :first-of-type::first-letter Selects a specific part of the element.

Best Practices

  • Style the generic element first.
  • Try to minimize the use of classes and id's on the html.
  • Try to use descendants and pseudo classes and elements when possible.
  • Avoid using id's for styling. They are useful for Javascript targetting.
