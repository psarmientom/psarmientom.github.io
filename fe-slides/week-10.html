Single Page React App

  1. Open the Terminal.
  2. Navigate to the directory where you have your class projects using cd. In my case I have them on my Github floder located inside the Documents folder.
    cd Documents/Github
  3. Once in the folder, create the app by typing npx create-react-app react-app
    (Where react-app is the name I assigned to the app. You can use a different one).
  4. After you get the "Happy Hacking!" message, the app is installed.
  5. Close the terminal.

Create the Github Repository

  1. Open Visual Studio Code.
  2. Select New window and open the folder with the recently created app.
  3. Select the Source Control Icon and click on "Publish Branch". This will create a repository with the same name on Github.
  4. Login to your Github account and make sure the repository has been created.
  5. If the repository is there, you can close Github for now.

Starting the React App

  1. Go back to Visual Studio
  2. Open the Visual Studio Terminal.
  3. Since you are already inside the folder, you don;t have to use CD. Type npm start
    A new browser window will open with the React logo.

Configuring an External Template

  1. Download the bs-basic zip file with the Bootstrap template.
  2. You can store and unzip the file in the same directory as your app, but not inside the app folder.
  3. Open an new window in Visual Studio and open the bs-basic folder. You need to have both projects open in different Visual Studio windows.

Preparing HTML content for React

There are a few things you need to change in the html in order to make it compatible with React. These are some of the things you need to look for:

  • css classes: Since class is a reserved javascript word, you need to replace all instances of class for className in all your the .js files you created.
  • inline styles: If you find inline styles on the html template and you cannot move them to a regular CSS file, you need to format it to work with React. If you have:
    style="height:50px" you need to change it into style={{height:"50px"}}. For multiple properties, separate the properties with a comma.
    style={{height:"1rem" , width:"1rem"}}
  • Hyphenated Words: You need to change them into camelCase words. file-size becomes fileSize. If the hypens are part of a regular css in "", you don't need to change them.
    class="home-content" is correct because in between quotes.

Other HTML errors

While working on the app, keep your eye on the Problems panel on Visual code and look for any HTML error. While your app will work locally with those errors, it will not deploy correctly until you fix them.

One important error to consider is the image alt tags. If you're using a decorative image just add and empty tag, but the tag must be present

alt=""

The Navbar

Now that we have all the content, we need to fix the link, to show only one content at a time and replace the main content with the one associated with the current link.

In order to do that, we need to install the react-router-dom which is a React library used to implement dynamic routing.

  1. Stop the server, by going to the Visual Studio terminal and type Control C After this, you should not be able to see your page anymore in the browser.
  2. Go back to the VS terminal and type
    npm install react-router-dom
  3. Restart your React app by typing npm start. You should be able to see your app on the browser again.

Customizing the Nav.js

Since we are using the React Router, we need to replace the <a> and the href, so it can work correctly.

  1. Go to the Nav.js file and import the Link from the react router at the top of the page, below the React import.
    import {Link} from "react-router-dom"
  2. replace all <a for <Link and </a> for </Link>
  3. Replace all href= for to=
  4. Replace the to="#" on line 11 to to="/"
  5. Replace the links to the .html files with links to the React apps.
    • index.html to /
    • team.html to /teams
    • about.html to /about
    • contact.html to /contact

At this point your app is not going to work, because we have to import the react router into the app.js as well.

Routing on the App.js

To be able to route the url correctly, we need to import a few Routing Functions

  1. Add import { HashRouter, Routes, Route} from "react-router-dom" after all your imports at the top of the page.
  2. We need to indicate that we are routing, inside the function, so we replace the React fragments.
    • <> for <HashRouter>
    • </> for </HashRouter>
    At this point your app should be back in the browser.
  3. Now we need to set the content that needs to be routed. The Nav and the footer are static and should remain in all pages, but the Home, Teams, About and Contact need to be re-routed.

Routing

  1. We need to wrap the 4 pages in a Routes tag
  2. Then we need to add the path and the element it should load. Your return code in the App.js should look like this:
  • <HashRouter>
  • <Nav />
  • <Routes>
  • <Route path={"/"} element={<Home />}
  • <Route path={"/teams"} element={<Teams />}
  • <Route path={"/about"} element={<About />}
  • <Route path={"/contact"} element={<Contact />}
  • </Routes
  • <Footer />
  • </HashRouter>

Your app should work locally now.

Publishing to Github

You have your app working locally, but you need to publish it online. Depending on the server, the steps are different, but for Github this is what you need.

  1. Go to your repo and select Settings > Pages
  2. On the Right panel, select master and Save. The message for "Your site is ready to be published at "your page URL" will display.
  3. Take note of the URL because you are going to use it on the next step.

Preparing to Build

So far, we have our application ready on our local environment, but we need to make it production ready. For Github we need to install one more thing.

  1. Open the package.json file. and add below the name line:
    "homepage": "https://username.github.io/test-app", replace the username with your Github username and test-app with the name of your repo.
  2. On the same file, scroll down to the Scripts section and in the line eject, add a comma at the end and on a new line add:
    • "predeploy": "npm run build",
    • "deploy": "gh-pages -d build"

Deploying to Github Pages

  1. Open Visual Studio Terminal and stop the server by typing Control C
  2. Type npm install gh-pages --save-dev
  3. Then type npm run build. Your files will be copied to the gh-pages branch on Github.

Publishing on Github.

When you publish a react app on Github it will be run from the gh-pages branch which will be created when you deploy your React app.

  1. Go to the repository and select Settings > Pages
  2. On the Right panel you should have master on the Source dropdown.
  3. Click and select the gh-pages branch and save. Your app will be published.
