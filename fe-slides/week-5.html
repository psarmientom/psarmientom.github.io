What is Javascript?

  • Javascript is the programming language of the web.
  • It is also called Vanilla Javascript.
  • It was created in 1995 to provide some interaction and load dynamic content in the browser.
  • At the beginning, it used to run only in the browser, but it has evolved that now it can run in both the browser and the server.
  • Today, Javascript can be used to create not only websites, but also full web applications with complex functionality, mobile and smartwatch applications and can be combined with many other languages.
  • Javascript is not related in any way to the Java programming language.
  • Javascript doesn't need to be compiled (translated from human-readable to machine code).

What does Javascript do?

  • It detects user interaction and provides a response.
  • It lets you create animations and interactive components.
  • Javascript lets you alter the structure of your HTML and CSS using an API called the DOM.

And what do API and DOM mean?

  • API stands for Application Programming Interface. It is sometimes called Web Services.
  • The function of an API is to serve as a link between 2 applications so they can talk to each other.
  • JavaScript when used on the frontend, interacts with browsers through a set of functions collectively known as Web APIs.
  • DOM stands for Document Object Model and it's a representation of the HTML and CSS.
  • On a website, the Javascript can interact with the HTML and CSS through the DOM.

The DOM (Document object Model)

  • The "View Source" is the HTML before it's processed by the browser.
  • The DOM is the representation of the page after all the css and javascript has been processed.
  • Each HTML tag is an element. And each Element is a DOM node. Then the browser creates the node tree.
  • The HTML does not change, but the DOM does.

Adding Javascript to your HTML file.

  • Create a js folder on your project to store all your javascript files. Javascript files have the extension .js.
  • In the HTML document, Javascript files can be added to the head tag, or right before the closing body tag, but depending on where you add them, you might have to use defer / async.
  • <script src="js./myjsFile.js">

Async, Defer or Nothing

Simple script tag

Added at the bottom, waits until all the HTML has been loaded.
Added on the head, it can stop loading the html until the script has been loaded.

Defer

Added on the head, waits until the HTML is loaded to execute the script. (preferred option).

Async

Added on the head, loads and executes scripts simultaneously. Recommended for when the scripts don't need the DOM.

Data Types

In Javascript there are 7 basic Data types which are different ways in which data is classified to tell the computer how to interpret it. They are also called primitives, because they represent a primitive value, that is not an object and has no methods.

  • Strings: They represent text data "I'm a string" and they always go between quotations.
    • Single or double quotes? It does not matter as long as you are consistent and pair them correctly.
    • And what about apostrophes? If you use double quotes, they don't represent a problem, but with single quotes you have to escape them. 'I\'m a string'
    • And quotes inside quotes? You have to escape it or use the other type of quote inside. 'This is a "quote"' or "This is a \"quote\""
  • Numbers: Represent Numeric data. You can make math operations with numbers.
    • + Addition. - Subtraction.
    • * Multiplication. / Division.
    • ++ Add one to a number. -- Subtract one to a number.
    • % Return the remainder after dividing 2 numbers.
    • NaN means "Not-a-Number" and shows up when JS doesw not have a way to interpret that value as a number.
  • Boolean: The keywords true or false. Usually used with if statement to make sure a contition is happening.
  • Null represents a Non-Value. Undefined represents anything that has not beed defined by JavaScript or us.
  • Symbols: Introduced in 2015 with ES6, symbols are used to identify object properties.
  • BigInt: Big Integrer and lets you create extremely large numbers that are not supported by the Number Data Type.

Values

  • The content of a string I'm a string is a value.
  • The content of a number 7 is a value.

Variables

  • A variable is a container for a value.
  • The value can be any kind of data (strings, numbers, elements from the DOM, functions).
  • We can use variables in all or parts of our script.
  • Variables can only take one value.
  • A variable must be declared before you can use it.

How do I declare a Variable?

const variableName = 'value'
  • variableName or identifier is the name you assign to your variable. It must follow the JavaScript syntax conventions.
  • "value" Is what you want your variable to be. It can be a primitive or an object.
  • = Assigns a value to your variable. (more on this later).
  • You can use any word except for some Javascript reserved words.

= and + in Javascript

In JavaScript these symbols have a slight different meaning depending on how they are used:

  • a = 1 does not mean that a is equal to 1, but that a has been assigned the value of 1.
  • 1 == 1 means that I'm comparing 1 to 1 and they are equal.
  • a != 1 means that a is NOT equal to 1.
  • 4 + 7 I'm adding 4 plus 7 because I'm dealing with 2 numbers.
  • "4" + 7 does not give the same result because we are dealing with a string and a number. In this case the + is used to concatenate both elements.
const variableName = 'value'

const - let - var

We see variables declared using these 3 methods. What's the difference?

var

  • This is the old way to declare a variable.
  • Var can be globally scoped or function scoped.
  • The can be re-declared and updated.

let

  • Introduced in 2015 with ES6.
  • let allows you to update the value of a variable.
  • Like const, cannot be re-declared.
  • They are block scoped.
  • Can be declared without being initialized.

const

  • Introduced in 2015 with ES6.
  • Represents a variable whose value cannot be updated.
  • Cannot be re-declared.
  • They are block scoped.
  • const variables must be initialized at the declaration time.

To initialize a variable is to give it a correct initial value.

Which one is better?

  • var is too flexible. It can be global, can be re-declared and re-assigned, so it's more prone to errors and leads to harder debugging.
    • The more control you have over your code, the lees chances of errors.
  • You can use either let or const.
  • There are countless arguments for either side.
  • let could be easier to use at the beginning, but const is less prone to errors.

Functions

A function is a block of reusable code that performs a task in a specific order.

We can say that an function is an object that can do something, not just a container for a value.

Declaring a Function

  • function functionName (optional parameters){
    // do something here
    }
  • function is the keyword that tells Javascript that you are declaring a function.
  • functionName is the name of the function and should follow JavaScript syntax conventions.
  • (parameters) represent variables that should be created by the function passing a given value.

Invoke, Run, Execute, Call

Many names to make a function work.

  • You need to declare a function before you can use it.
  • In order for the function to work, it needs to be executed somewhere in the document and outside the function.
  • function functionName (optional parameters){
    // do something here
    }
    functionName(arguments)
  • (arguments) are the values used for each parameter when you call a function.
  • If you use parameters, you have to declare the arguments when you call the function.

The Return statement

  • The return statement returns a specific value fom the function to the function caller.
  • function functionName (){
    return "something"
    }
    functionName()
  • It should be used at the end of the function because nothing else will execute after.
  • It can return primitive values and objects.

Function expression

This is another way of declaring functions.

  • You declare a variable, then assign a function without a name.
  • const myFunctionExpression = function () {
    console.log('This is a function expression!')
    }

Arrow Functions

Arrow functions are an alternate way of declaring functions introduced in 2015 with ES6

  • const myArrowFunction = function (par1, par2) {
    console.log('This is a function expression!')
    }
  • const myArrowFunction = (par1, par2) => {
    console.log('This is a function expression!')
    }
  • Without parameters: const myArrowFunction = () => {}
  • Or: const myArrowFunction = _ => {}
  • In case you just want a return statement you can skip the {}
  • cosnt myArrowFunction = (par1, par2) => par1 + par2

Objects

An object is a collection of related data, in the form of "key:value" pairs.

const anObject = {
key1: 'value1',
key2: 'value2',
key3: 'value3',
// ... }
const myDog = {
name: 'Bernie',
color: 'brown and white',
breed: 'St. Bernard',
// ... }
  • Each key: gives a reference to a value.
  • Key names have the same constraint as naming variables. (no spaces, no starting with numbers, etc).
  • If you use a different syntax, then it becomes an Invalid Identifier, and you have to use quotes 'dog name'.
  • The value can contain anything that is valid in Javascript (strings, numbers, booleans, functions, arrays, objects.
  • Object keys are called properties.
  • name, color, breed, are the properties ob the object myDog.

Getting the value of a property

You can get to the values of a property 2 ways: the dot notation or the bracket notation.

  •                             const myDog = { 
    
                                name: 'Bernie', 
    
                                color: 'brown and white', 
    
                                breed: 'St. Bernard', 
    
                                weigth: '150lb' 
    
                              }
  • Dot Notation: const dogName = myDog.name
  • Bracket Notation: const dogName = myDog['name']
  • Use dot notation all the time, unless you are using an Invalid Identifier, then you use Bracket Notation.
  • In order to avoid using Bracket Notation, try to name your properties correctly, using proper Javascript syntax.
