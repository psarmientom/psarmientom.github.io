HypeText Markup Language

What is a HTML?

* * *

  • It's a Markup Language
  • Not a programming Language
  • Used for creating webpages
  • Human Readable
  • Defines the structure of webpages
  • It encapsulates, or “marks up” data within HTML tags

Markup Langage

  • Presentational Language
  • No logic or algorithm
  • Used to define the structure of data
  • HTML, CSS, XML

Scripting Language

  • Subcategory of Programming Languages
  • Integrates and communicates with other languages
  • Easier to learn
  • Used in web applications, games, multimedia
  • JavaScript, PHP

Programming Language

  • Set of Instructions
  • Logic
  • Communicate with hardware with a set of instructions
  • Used for software, drivers, etc.
  • C, C++, Java

Opening Tag

Anatomy of an HTML page

  • Doctype: Tells the browser the file type
  • Html: Indicates anything between them is html code, language and reading direction
  • Head: Contains page information (title, links to external files etc.)
  • Body: Main content of the webpage. Contains all the other html tags
<!DOCTYPE html>
<html lang="en" dir="ltr">
    <head>
      <!-- Here there is page information for the browser, links to external files needed. Not visible on the actual webpage.-->
      <meta charset="utf-8">
        <title>
        This is the page Title on the top of the browser
        </title>
    </head>

      <!-- This is what's visible on the website -->
      <body>
        <section class="bg-primary">
            <div class="wrap">
                <h2> This is a Heading</h2>
            </div>
        </section>
      </body>
  </html>

Web links

  • Links to other pages on your website
  • <a href="myPage.html">
  • Links to other websites
  • <a href="http://otherPage.html">

Anchor Links

  • Links to other sections of the same page
  • identify the link using the #
  • <a href="#newSection">
  • And the target section
  • <a name="newSection">

Email Links

  • Opens your default email client
  • myEmail@email.com
  • <a href="mailto:myEmail@email.com">
  • Vulnerable to spam bots getting your email address.

Relative links

<a href="myPage.html">

  • This is the location of the file relative to the page your're working on
  • The file belongs to your site
  • They share the same URL

Absolute links

<a href="http://notMySite.com/Page.html">

  • This is the location of the file on the internet
  • Does not belong to your site
  • You have to include the full web address, so it can be located

Local Links

<a href="/Users/username/Documents/GitHub/username.github.io/myPage.html">

  • This is the location of the file in your computer
  • This will work fine while you are working in your local copy
  • When you upload the project to Github, the link will be broken because other people will not be able to access the file in your computer

Web Images

As a general rule of thumb, we like our product photos to be a maximum 80-150 KB, and for large homepage hero images to be a maximum of 250-350 KB.

Image file size is important for site speed's and performance.

JPG

  • A compressed image format standardized by the Joint Photographic Experts Group (JPEG).
  • It significantly reduces the file size of images
  • Good for photographs
  • Some information is lost in the process

PNG

  • Image file stored in the Portable Network Graphic (PNG) format
  • Commonly used to store web graphics, digital photographs.
  • Support Transparency
  • Does not degrade the image quality
  • Bigger image size than jpg

SVG

  • SVG stands for Scalable Vector Graphics
  • Used to define vector-based graphics for the Web
  • Images are scalable
  • Graphics do NOT lose any quality if they are zoomed or resized

GIF

  • GIF Stands for Graphics Interchange Format
  • Does not degrade the image quality
  • Can include a maximum of 256 colors
  • Better suited for buttons and banners that do not require a lot of colors.

Semantic Markup

* * *

  • It is HTML code structured for meaning
  • Using HTML elements according to their intended purpose
  • HTML elements are not used for visual presentation
  • header, footer, nav, section, article, aside, blockquote

Why is it important?

  • Search Engines do not see the webpage styles, they go by content
  • Establishes a clear hierarchy
  • It is easier to understand for people who use screen readers (Accessibility)
